# HonestKeyAudit

**A defensive RSA & ECC weak-key auditor — find weak cryptographic keys before an attacker does, and understand *why* they are weak.**

Author: **Cristian Vitorino Rodrigues da Cunha** — independent researcher
Scientific basis: *A Silver-Phase Barrier for Integer Factorization* — DOI [10.5281/zenodo.14553556](https://doi.org/10.5281/zenodo.14553556)

> **Honest scope.** These detectors find *known* weak-key classes to **protect** the owner. They do **not** break strong RSA/ECC with well-generated keys — and that limit is itself a theorem (the Barrier, below). Defensive value is twofold: catch keys already issued that fell into a weak class, and guard freshly generated keys against falling into one.

---

## Why this exists

Most people and small organizations **never audit their keys**. A key generated by a flawed RNG, a buggy library, or a low-entropy device can fall into a *known* weak class that is trivially breakable — and sit in production for years. Good free tools exist for parts of the RSA side (batch-GCD, ROCA detection), but **ECC auditing is poorly packaged and rarely accessible**. HonestKeyAudit brings RSA and ECC checks into one clear tool with plain-language reports.

## What it checks

### RSA — `src/rsa_audit.py`
| Check | Weak class |
|---|---|
| Fermat | primes too close (`\|p−q\|` small) → factorable in a few steps |
| Batch-GCD | shared prime across a key corpus (bad PRNG / entropy reuse) |
| mod-M bias | ROCA-style restricted residue class (fast heuristic) |
| **ROCA** (CVE-2017-15361) | Infineon RSALib structure — the famous 2017 vulnerability (billions of devices) |
| p−1 smooth | Pollard p−1 vulnerable (guard mode) |
| **Wiener** | small private exponent `d` → key fully broken from `(e, N)` |

Verified: **AUC = 1.00, zero false positives** on synthetic corpora.

### ECC — `src/ecc_audit.py`
| Check | Weak class |
|---|---|
| On-curve | public point not on the stated curve (invalid-curve exposure) |
| Standard curve | non-standard parameters (weak/backdoor risk) |
| Subgroup | point of small order (small-subgroup / cofactor) |
| Anomalous | `#E = p` → Smart's attack (polynomial-time discrete log) |
| MOV | small embedding degree → pairing attack |
| **ECDSA nonce reuse** | repeated nonce → **private key leaks** (the Sony PS3 / wallet class) |

Verified: valid keys pass; off-curve, non-standard, and nonce-reuse cases are caught — with private-key recovery demonstrated to prove why reuse is fatal.

## Beyond per-key: generator-anomaly auditing (the differentiator)

Every other tool checks **each key** against **known** weak classes. `src/gen_anomaly.py`
checks a **population** of keys against the distributions proper random generation should
follow — flagging a **biased generator** that per-key checks (ROCA included) miss, before it
becomes the next ROCA. Two lenses, both computable from the public modulus:

- **Residue uniformity** (χ² of `N mod m`) — catches modular RNG bias.
- **Silver-phase / Hecke equidistribution** (KS test on `φ(N) = frac(log_δ N)`) — the object
  from our published no-go theorem; catches a bias on the *logarithmic axis* (e.g. low-entropy
  key sizing) that the residue lens and per-key checks do **not** see.

Verified: a good generator passes; a modular-biased generator is caught by the residue lens; a
size-biased generator is caught only by the silver-phase lens — while per-key ROCA flags neither.

**Continuous monitoring / drift detection (`src/drift_monitor.py`)** — the productized form. Fingerprint
a *trusted* generator's key population, then check every new batch for **drift**: silent generator
change, entropy degradation, or **supply-chain contamination** (foreign keys mixed into a legit
population). Validated: no false alarm on the same generator; drift detected on a generator swap and on
20% contamination — flagging foreign keys individually (per-key certainty via the FIPS/OpenSSL
top-2-bits floor) plus two-sample tests on the silver phase, leading value, and residues.

## Scale study: batch-GCD (`src/batchgcd.py`)

At internet scale, a small fraction of RSA keys share a prime factor (weak RNG / low
entropy) — and `gcd(N1, N2)` then reveals **both** private keys. This is how *Mining your
Ps and Qs* (2012) found hundreds of thousands of breakable keys in the wild. Our engine
uses **Bernstein's batch-GCD** (product tree + remainder tree, `O(n log²n)`), scaling to
millions of moduli. Install `gmpy2` for a 10–100× speedup (≈25k moduli/s → millions in
minutes). Ingests a moduli text file, a folder of PEM/DER certs, or live GitHub SSH keys:

```bash
python src/batchgcd.py selftest                    # proves it finds injected shared factors
python src/batchgcd.py moduli.txt                  # one N per line (dec or 0x-hex)
python src/batchgcd.py --pems /folder/of/certs
python src/batchgcd.py --github user1 user2 ...     # real SSH keys (set GITHUB_TOKEN for scale)
```

Public research datasets for a real measurement study: Rapid7 Open Data / scans.io, Censys,
GitHub SSH keys. Yield is empirical — modern keys are mostly clean, but at scale a real
fraction of breakable keys still appears.

## Two modes

- **audit** — scan a corpus of *public* keys/signatures; report which are weak.
- **guard** — verify a *freshly generated* key does not fall into any known weak class before deployment.

## Quick start

```bash
pip install -r requirements.txt

# Web app — paste a certificate/key in your browser, get a report (runs locally):
python src/webapp.py            # open http://localhost:8000

# Or from the command line — point it at a certificate/key file or a whole folder.
# Reads PEM/DER X.509 certs, RSA/EC public keys, and OpenSSH keys.
python src/keyaudit.py /path/to/cert.pem
python src/keyaudit.py /path/to/keys_folder/     # scans all keys + cross-key batch-GCD
```

Low-level / research interfaces:

```bash
# RSA
python src/rsa_audit.py selftest              # self-test (AUC=1.00, zero FP)
python src/rsa_audit.py audit examples/sample_moduli.txt
python src/rsa_audit.py guard <p> <q>

# ECC
python src/ecc_audit.py selftest
python src/ecc_audit.py pubkey P-256 0x<Px> 0x<Py>
```

## The research behind it

HonestKeyAudit is the applied side of the **Projeto Prata** research programme,
which followed a hand-built spreadsheet of prime "voids" to the frontier of the
Riemann Hypothesis, with honest computational verification at every step. Its
central publishable result is **the Barrier**: recovering the silver-coordinate
phase of a factor is polynomial-time equivalent to factoring — a rigorous *no-go*
theorem, generalized to show it is a property of the *scale* of any continuous
lens, not of the silver ratio. That negative result is *why* this auditor is
honest about its limits: there is no hidden geometric shortcut, so it detects only
real, known weak classes. See the paper: DOI [10.5281/zenodo.14553556](https://doi.org/10.5281/zenodo.14553556).

## Ethics

Defensive use only. Run on **your own** keys, synthetic keys, or public research
datasets with responsible disclosure. Never against third-party keys without
authorization. For the ECDSA nonce-reuse detector, the goal is detection and
responsible disclosure of exposure — never harvesting keys that control funds.
